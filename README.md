# Skills

My Claude Code skills for fullstack web development.

## Engineering

| Skill | Description |
|---|---|
| [/plan](./plan/SKILL.md) | Phase 1 — Research + planning. Creates a GitHub Issue before touching code. |
| [/build](./build/SKILL.md) | Phase 2 — Implementation. Backend first, type-check at each layer, then frontend. |
| [/cleanup](./cleanup/SKILL.md) | Between build and ship — service layer extraction, duplication scan, naming pass. |
| [/ship](./ship/SKILL.md) | Phase 3 — Playwright E2E, quality gate, commit, push, PR. |
| [/deepen](./deepen/SKILL.md) | Periodic whole-codebase architecture review. Run every few weeks, not after each feature. |
| [/grill-me](./grill-me/SKILL.md) | Stress-test a plan by walking the full decision tree before any code is written. |

## Design

| Skill | Description |
|---|---|
| [/impeccable](./impeccable/SKILL.md) | UI design and frontend polish — critique, shape, animate, colorize, and more. |
| [/design-inspiration](./design-inspiration/SKILL.md) | Load design context (DESIGN.md, references, notes) before any UI work. |
| [/design-references](./design-references/SKILL.md) | Set up design direction from reference images. Run once per project. |

## Installation

```bash
git clone https://github.com/Jahazel/skills ~/Code/skills
cd ~/Code/skills && ./sync.sh
```

This copies all skills into `~/.claude/skills/` where Claude Code can read them.
